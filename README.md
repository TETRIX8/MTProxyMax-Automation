# MTProxyMax Automation & API Manager

Данный репозиторий содержит набор скриптов для автоматического развертывания и управления Telegram прокси-серверами на базе **MTProxyMax**. Система реализует разделение на **Управляющий сервер (Master)** и **Рабочие серверы (Worker)**, а также включает автоматизированный пул ключей с API доступом.

## 🚀 Быстрая установка (Bash Link)

Для автоматического развертывания всей инфраструктуры на **любом новом сервере**, используйте следующую команду:

```bash
curl -sL https://raw.githubusercontent.com/TETRIX8/MTProxyMax-Automation/main/setup.sh | bash
```

### Режимы запуска:
1.  **Интерактивный:** Запустите команду без аргументов, и скрипт запросит IP, пользователя и пароль удаленного сервера.
2.  **Автоматический:** Передайте параметры сразу (IP и Пароль):
    ```bash
    curl -sL https://raw.githubusercontent.com/TETRIX8/MTProxyMax-Automation/main/setup.sh | bash -s <IP_СЕРВЕРА> <ПАРОЛЬ_ROOT>
    ```

## ⚙️ Основные возможности

*   **Авто-развертывание:** Развертывание всей инфраструктуры (MTProxyMax + API + Автоматизация) одной командой.
*   **Умный пул ключей:** Система заранее генерирует 100 тестовых и 100 обычных ключей каждый день в 7:00 утра.
*   **Авто-пополнение:** Если количество свободных ключей в пуле опускается ниже 20, система автоматически генерирует еще 20 штук "про запас".
*   **Готовый API:** Легкий FastAPI сервер для мгновенной выдачи ключей (возвращает готовую ссылку `https://t.me/proxy?...`).
*   **Лимиты:** Автоматическая установка срока действия ключей (1 день для тестов и любой период для обычных).

## 📡 API Документация (Примеры для вашего сервера)

API сервер работает на порту `8000`.

### Авторизация
Все запросы должны содержать заголовок:
`Authorization: Bearer MTProxyMaxSecretToken123`

### 1. Получить тестовый ключ (на 1 день)
Выдает ключ из пула тестовых со сроком действия **1 день**.

**Пример запроса:**
```bash
curl -X GET "http://38.244.138.57:8000/get-test" -H "Authorization: Bearer MTProxyMaxSecretToken123"
```

**Пример ответа:**
```json
{
  "link": "https://t.me/proxy?server=38.244.138.57&port=443&secret=...",
  "label": "test_1775396181"
}
```

### 2. Получить обычный ключ (на любой период)
Выдает ключ из пула обычных с заданным периодом.

**Пример запроса (на 30 дней):**
```bash
curl -X GET "http://38.244.138.57:8000/get-regular?label=client_01&period=+30days" -H "Authorization: Bearer MTProxyMaxSecretToken123"
```

**Пример запроса (на 1 год):**
```bash
curl -X GET "http://38.244.138.57:8000/get-regular?label=client_01&period=+1year" -H "Authorization: Bearer MTProxyMaxSecretToken123"
```

## 🛠 Как это работает внутри

### Логика пула (`worker.sh`)
Скрипт управляет файлами `/opt/mtproxymax/pool_test.conf` и `/opt/mtproxymax/pool_regular.conf`.
1.  **7:00 AM:** Cron запускает `worker.sh --daily`, который наполняет файлы до 100 чистых секретов.
2.  **Запрос API:** Скрипт `pool_manager.sh` берет верхнюю строку из файла, удаляет её и регистрирует в MTProxyMax.
3.  **Check:** Если в файле осталось < 20 строк, запускается фоновое пополнение (`--check`).

### Структура файлов на Worker
*   `/opt/mtproxymax/api_server.py` — FastAPI сервер.
*   `/opt/mtproxymax/worker.sh` — Логика генерации и пополнения пула.
*   `/opt/mtproxymax/pool_manager.sh` — Интерфейс между API и MTProxyMax.
*   `/opt/mtproxymax/pool_*.conf` — Файлы с готовыми секретами.

## 🛡 Безопасность
*   Пароли для SSH не хранятся в открытом виде после установки.
*   API защищен статическим токеном (рекомендуется сменить `API_TOKEN` в `api_server.py`).
*   Секреты генерируются через `openssl rand -hex 16`.
